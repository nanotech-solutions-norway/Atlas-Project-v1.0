# Outcome Selector

Purpose: capture a visitor’s use case, segment, and package intent early so Atlas can route them through Orient → Explain → Fit → Prove → Act.

## Launch fields
- Use case or need
- Segment / role
- Package intent
- Team size band
- Governance sensitivity

## Operational rule
The selector must update CTA labels, proof adjacency, and recommended pages without hard-coded platform assumptions outside the CMS and routing model.
